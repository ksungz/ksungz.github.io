## 문제 인식

팀에서 AI 코딩 도구를 본격적으로 도입하면서, 각 저장소마다 독립적으로 규칙(Rules)과 스킬(Skills) 파일을 관리하는 것의 한계가 드러났습니다.

- 7개 이상의 프로젝트 저장소에서 동일한 AI 설정을 각각 관리 → 설정 파편화
- 한 저장소에서 개선한 규칙이 다른 저장소에 반영되지 않는 비일관성
- 신규 팀원의 AI 도구 온보딩 시 어떤 설정을 써야 하는지 혼란

## 해결 전략

AI 설정 파일들을 단일 저장소에서 중앙 관리하고, CLI 명령으로 각 프로젝트에 자동 배포하는 NPM 패키지를 개발했습니다.

### 핵심 기능

- **중앙 집중 관리** — 모든 AI 규칙·스킬을 하나의 저장소에서 버전 관리
- **CLI 배포** — 명령어 한 번으로 지정된 프로젝트들에 설정 파일을 동기화
- **선택적 배포** — 프로젝트별로 필요한 설정만 선택적으로 적용 가능
- **버전 관리** — 설정 변경 이력을 추적하고 롤백 가능
- **MCP 서버 설정** — 팀 공용 MCP 서버 설정도 함께 관리·배포

## 관리하는 설정 종류

| 설정 유형 | 용도 | 예시 |
|---------|------|------|
| Rules | AI 코딩 도구의 행동 규칙 | 커밋 컨벤션, 코드 리뷰 기준, Git 워크플로우 등 |
| Skills | 특정 작업을 자동화하는 절차 | PR 생성, 마크업 리뷰, 커밋 메시지 생성 등 |
| MCP 설정 | 외부 서비스 연동 설정 | Confluence, Jira, Bitbucket 연동 등 |

## 성과

- 7개 이상의 프로젝트 저장소에서 **일관된 AI 개발 환경** 유지
- 규칙 업데이트 시 **전체 프로젝트 동시 반영** 가능 (수작업 제거)
- 신규 팀원의 AI 도구 온보딩 시간 대폭 단축
- 팀 내 AI 활용 사례가 자연스럽게 축적·공유되는 선순환 구조 형성
