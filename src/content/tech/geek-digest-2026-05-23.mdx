TalkMode를 처음 봤을 때 "또 음성 챗봇이네" 하고 넘길 뻔했다. 근데 Baryon Labs(서울) 팀이 만든 이 도구, 뜯어보면 결이 다르다. 캘린더를 읽고, 메일 초안을 쓰고, 시스템 상태를 쿼리하는 — 단순 응답이 아니라 실제 Tool 호출이 되는 Agent-OS 구조다. Apple Silicon 온디바이스에서 시선 추적(Gaze)으로 '눈을 돌리면 대화 종료'를 구현한 게 특히 이상했다. 이상하다는 게 나쁜 말이 아니라, 프론트 카메라 데이터가 외부로 안 나가는 채로 물리적 멀티모달 인터랙션을 만든다는 게 — 음성 AI가 '모바일 비서' 방향이 아니라 개발자 워크플로우에 붙는 쪽으로 갈 수도 있겠다는 느낌을 준다. Claude CLI / Codex CLI를 백엔드로 쓰는 구조도 그렇고, 터미널 문화 위에 음성 레이어를 얹는 시도 자체가 낯설다. AGPL-3.0 오픈소스라 직접 파볼 수 있다는 것도 플러스.

---

두 번째는 읽다가 좀 불편해진 글이다. AI 코딩 에이전트에서 같은 모델, 비슷한 토큰 수(300K vs 257K)인데 비용이 $0.0096 대 $0.4455 — 46배 차이가 난다. 원인은 Anthropic prefix caching의 TTL이 5~10분이라는 것, 그리고 context compaction이 한 번 터지면 캐시가 전면 무효화된다는 것. 매 턴 전체 transcript를 쌓아서 보내면 캐시 히트할 때는 싸지만, compaction 한 번에 $0.44짜리 청구서가 날아온다. 저자는 이걸 '캐시 복권'이라고 불렀는데 적확하다. 대안으로 제시한 structured state — 핵심 상태만 추출해서 전송하는 방식 — 은 44턴 세션에서 토큰을 3,777개에서 740개로 80.4% 줄였다. 턴당 단가는 약간 오르지만 스파이크가 없고 세션 재시작 비용도 3K 고정 vs 누적 17K+라 예측 가능하다. 요는 토큰을 줄이는 게 목표가 아니라, 캐시가 깨졌을 때도 구조가 버텨야 한다는 거다. 에이전트를 조금이라도 길게 돌려본 사람이라면 이게 아키텍처 얘기라는 걸 바로 안다.
