## 도입 배경

컨벤션 미준수, 접근성 속성 누락 등 반복적으로 발생하는 패턴의 리뷰를 자동화하고 싶었습니다. 사내에서 제공하는 AI PR Review Agent를 팀 저장소에 도입하기로 했습니다.

## 담당 작업

사내 제공 AI PR Review Agent를 팀에서 관리하는 저장소들에 적용했습니다. 직접 개발한 것이 아니라, 각 저장소의 CI 파이프라인 yml 파일을 수정해 Agent를 연동하고 동작 방식을 설정한 작업입니다.

- 8개 저장소의 빌드 파이프라인 yml 파일에 AI Agent 스텝 추가
- PR 생성 시 자동 트리거 및 On-Demand 수동 트리거 방식으로 설정
- 저장소별 파일 필터링 규칙 설정 (SCSS, HTML 마크업 파일 위주)
- 팀의 코딩 컨벤션, 접근성 기준, BEM 네이밍 규칙을 리뷰 기준으로 반영

## 리뷰 대상 항목

| 카테고리 | 점검 항목 예시 |
|---------|--------------|
| 코드 품질 | 불필요한 코드, 중복 선택자, 미사용 변수 |
| 접근성 | ARIA 속성, 시맨틱 태그, alt 텍스트, 키보드 네비게이션 |
| 컨벤션 | BEM 네이밍, SCSS 네스팅 깊이, 클래스 접두사 |
| 성능 | 불필요한 !important, 과도한 셀렉터 특이성 |

## 성과

- 반복적인 컨벤션 리뷰 코멘트가 자동화되어 리뷰어의 부담 감소
- PR 리뷰 대기 시간 단축 — AI가 즉시 1차 리뷰 수행
- 접근성·컨벤션 관련 이슈의 사전 탐지율 향상
